<task type="auto" tdd="true">
  <name>Task 1: Implement releaseExpiredReservations service</name>
  <files>app/lib/sweeper.ts</files>
  <behavior>
    - Find all reservations where status = 'pending' AND expiresAt < now().
    - For each reservation, atomically:
      1. Decrement Inventory.reservedQty by reservation.qty (WHERE productId and warehouseId match).
      2. Set Reservation.status = 'released'.
    - Track success and failure per reservation.
    - Return object: { released: number, failed: number, errors: [{ id: string, error: string }] }.
    - Ensure confirmed reservations are never touched.
  </behavior>
  <action>
    Create `app/lib/sweeper.ts`. Implement `releaseExpiredReservations` using Prisma. Use a loop with per-reservation transactions to ensure a single failing update doesn't block other eligible reservations from being released. 
    
    Per D-14, the query should leverage the index on [status, expiresAt].
  </action>
  <verify>
    <automated>npm test -- --filter=sweeper</automated>
  </verify>
  <done>
    Service correctly releases multiple expired reservations, restores inventory, and handles partial failures.
  </done>
</task>
